// What's Included

Included with every VM.
No tiers, no add-on surprises.

These aren't premium features — they're the baseline. Every VM on every plan gets all of this.

[cpu]
Dedicated vCPU
Your CPU cores are reserved for you. We don't overcommit CPU across VMs on the same host. No noisy-neighbour CPU contention during peak hours.
Pinned cores · not shared
[nvme]
NVMe SSD Storage
All VM storage is NVMe — not SATA SSD, not spinning disk. Significantly faster random I/O for databases, high-traffic apps, and anything doing heavy disk work.
NVMe only · no HDD fallback
[ip6]
IPv6 /64 Allocation
Every VM gets a /64 IPv6 prefix — over 18 quintillion addresses. Dual-stack on paid plans (IPv4 + IPv6). IPv6-only Micro plan saves KES 200/month.
KIXP-peered · ~2ms to Kenyan ISPs
[ssh]
SSH Key Authentication
Password login is disabled by default. We configure SSH key auth during provisioning. Your public key goes in, you connect from day one — no password reset flow.
Password auth off · keys only
[rdns]
Reverse DNS (PTR)
Set your own PTR record for your IPv4 and IPv6 addresses. Essential if you're running a mail server. We update it on request — usually same day.
Customer-controlled · no extra charge
[dns]
Anycast DNS Hosting
Host your domain DNS on our anycast infrastructure. DNSSEC supported. Sub-50ms resolution across Kenya via KIXP. Free with every VM plan.
DNSSEC · free on all plans
[root]
Full Root Access
You own the OS. Install anything, modify anything, run any workload. Docker, K3s, Podman, custom kernels — no restrictions on what you run inside your VM.
sudo root · no agent, no backdoor
[snap]
Pre-Change Snapshots
Before any upgrade, plan change, or significant OS modification, we take a snapshot. Not sold as a feature — it's just what responsible operators do.
Before upgrades · at no charge

Network Monitoring
We monitor the network and VM hosts 24/7. If the host has a problem, we know before you do. Status at status.pylon.digital — always current.

// FAQ

VM questions.

Can't find your answer? Email us or check the full FAQ.

What does "no overselling" actually mean?+

When you buy a Standard VM with 2 vCPU and 4 GB RAM, those resources are reserved for you on the host server. We don't allocate more vCPUs across VMs than the host physically has. You will never experience CPU throttling because another customer on the same host is running a heavy batch job.

Many cheap VPS providers oversell — they sell more resources than they have, betting customers won't all use their allocation at once. We don't do this.

Can I upgrade my plan later without losing data?+

Yes. Send us a message and we'll schedule an upgrade window — usually within 24 hours. Upgrades require a brief VM reboot (typically under 5 minutes). We take a snapshot before any changes so your data is safe. Downgrades are possible if your disk usage fits within the lower plan's storage allocation.

How do I set my reverse DNS / PTR record?+

Email or WhatsApp us with your IP address and the hostname you want the PTR to resolve to. We update it typically the same day. This is free on all plans — essential if you're running a mail server, as most receiving mail servers check that your IP's PTR matches your mail domain.

Can I run Docker or Kubernetes on a Pylon VM?+

Yes — full root access means no restrictions. Docker, Docker Compose, Podman, K3s, RKE2 all work fine. For a lightweight Kubernetes cluster with K3s, we recommend Standard (single node) or two Premium VMs (HA control plane). We can help with network topology for multi-VM clusters if you need it.

What happens if my VM goes down?+

First, check status.pylon.digital. If it's a network or host issue, our NOC is already aware and working it. For network-level outages, our on-call engineer is reachable 24/7 via the emergency contact in your handover document.

If the issue is with software on your VM (not the host), that's within the self-managed scope. We can offer paid out-of-hours sysadmin support if needed — email us to arrange.

Is there a bandwidth or egress limit?+

No hard monthly cap on standard plans. We operate on fair-use — most customers never come close to causing issues. If your VM consistently uses very high egress (e.g. a high-traffic CDN origin or video streaming), talk to us upfront and we can arrange a custom plan.

Can I use my VM for a mail server?+

Yes. Set up Postfix/Dovecot (or any MTA) as you normally would. For good deliverability, request a PTR record matching your mail domain, configure SPF/DKIM/DMARC, and ensure you're not on any blacklists before starting. Our IPs are clean — we actively monitor abuse and don't permit spam under our AUP.

How is my VM backed up?+

VMs are not backed up by default — they're self-managed. We recommend enabling our daily backup add-on (KES 150/mo) for any production VM. This creates an off-site daily snapshot with 7-day retention, restorable on request.

We always take a manual snapshot before plan upgrades or changes at no charge. For application-level backup (database dumps, file exports), that's your responsibility.

Can I cancel any time?+

Yes. Month-to-month, 30 days' notice. No penalties, no lock-in. We'll help you export your data and confirm the shutdown date. If you need to leave in a hurry, we'll work with you on timing — 30 days is the norm, not a rigid rule for genuine emergencies.
